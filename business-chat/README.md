# Business Chat – Платформа безопасного обмена сообщениями

Полнофункциональное бизнес-приложение для обмена сообщениями, построенное на Next.js, PostgreSQL и современных веб-технологиях. Включает иерархическую организацию чатов, обмен файлами, реакции с эмодзи, сквозное шифрование, админ-панель и готовность к мобильным устройствам.

## Возможности

- **Иерархическая структура чатов**: Организация чатов по отделам, командам и группам с поддержкой родительских/дочерних отношений.
- **Богатые сообщения**: Текст, эмодзи, реакции, вложения (изображения, документы, видео, аудио) с локальным хранением файлов.
- **Сквозное шифрование**: Опциональное шифрование сообщений и файлов (поля `isEncrypted`, `encryptedContent`).
- **Управление пользователями**: Регистрация только через админку, аутентификация, доступ на основе ролей (Пользователь, Администратор, Супер-админ).
- **Профиль пользователя**: ФИО, отдел, должность, день рождения, аватар, статус в чате.
- **Админ-панель**: Управление пользователями, создание/редактирование иерархии компании и отделов, флажок активности, экспорт и очистка истории сообщений.
- **Пересылка сообщений**: Возможность переслать любое сообщение в другой чат с сохранением вложений.
- **Комментирование сообщений**: Ответ на конкретное сообщение с созданием цепочки обсуждения.
- **Редактирование сообщений**: Возможность редактировать отправленное сообщение в течение 5 секунд после отправки (только для отправителя).
- **Рассылка сообщений**: Отправка одного сообщения нескольким пользователям одновременно.
- **Индикаторы прочтения/доставки**: Отображение статуса сообщения (отправлено, доставлено, прочитано).
- **Настройка подключения к серверу**: Страница `/settings` для указания URL сервера и учётных данных.
- **Поддержка мобильных устройств**: Сборка нативных приложений для Android и iOS через Capacitor.
- **Обновления в реальном времени**: Живой обмен сообщениями с плавным интерфейсом.
- **Загрузка файлов**: Локальное хранение файлов в `public/uploads` с уникальными именами (UUID).
- **Уведомления о днях рождения**: Автоматические предупреждения за два дня до дня рождения пользователя (отправляются в системный чат).

## Технологический стек

- **Фронтенд**: Next.js 15, TypeScript, Tailwind CSS, React
- **Бэкенд**: Next.js API Routes, NextAuth.js, Prisma ORM
- **База данных**: PostgreSQL (через Prisma)
- **Хранилище файлов**: Локальное (папка `public/uploads`)
- **Аутентификация**: NextAuth с учётными данными
- **Шифрование**: bcryptjs, опциональное сквозное шифрование
- **UI-компоненты**: Lucide React, Sonner toasts, Emoji Picker React
- **Мобильная сборка**: Capacitor (Android, iOS)

## Начало работы

### Предварительные требования

- Node.js 18+ и npm
- База данных PostgreSQL (локальная или удалённая)
  - Рекомендуется использовать Docker Compose (файл `docker-compose.yml` в корне проекта)

### Установка

1. Клонируйте репозиторий:
   ```bash
   git clone <url-репозитория>
   cd business-chat
   ```

2. Установите зависимости:
   ```bash
   npm install
   ```

3. Запустите PostgreSQL (если не запущен):
   ```bash
   docker-compose up -d
   ```
   Или установите PostgreSQL вручную и создайте базу данных `business_chat`.

4. Настройте переменные окружения:
   Создайте файл `.env.local` в корне со следующим содержимым:
   ```env
   DATABASE_URL="postgresql://пользователь:пароль@localhost:5432/business_chat"
   NEXTAUTH_SECRET="ваш-секретный-ключ-минимум-32-символа"
   NEXTAUTH_URL="http://localhost:3000"
   ```
   Замените `пользователь:пароль` и `business_chat` на свои данные.

4. Настройте базу данных:
   ```bash
   npx prisma generate
   npx prisma db push
   ```

5. Запустите сервер разработки:
   ```bash
   npm run dev
   ```

6. Откройте [http://localhost:3000](http://localhost:3000) в браузере.

### Создание первого администратора

По умолчанию регистрация пользователей отключена. Чтобы создать первого администратора, выполните следующие шаги:

1. Откройте Prisma Studio:
   ```bash
   npx prisma studio
   ```

2. Перейдите на вкладку `User` и создайте нового пользователя со следующими данными:
   - `email`: admin@example.com
   - `password`: (хэшированный пароль, можно сгенерировать через [bcrypt генератор](https://bcrypt-generator.com/))
   - `role`: `SUPER_ADMIN`
   - `fullName`: Администратор
   - `department`: IT
   - `position`: Главный администратор
   - `isActive`: true

3. Войдите в систему с этими учётными данными.

## Использование

### Чат

- **Создание чата**: В левой панели нажмите «+ Новый чат», выберите тип (личный, групповой, канал) и участников.
- **Отправка сообщений**: Введите текст, добавьте эмодзи через палитру (кнопка 😀), прикрепите файлы (кнопка 📎).
- **Реакции**: Наведите на сообщение и нажмите на иконку «+», чтобы добавить реакцию.
- **Пересылка**: Нажмите на меню сообщения (три точки) и выберите «Переслать».
- **Рассылка**: В окне чата нажмите кнопку «Рассылка», выберите пользователей и отправьте сообщение.

### Профиль

- **Редактирование профиля**: Нажмите на свой аватар в правом верхнем углу → «Профиль».
- **Смена статуса**: Выберите статус (онлайн, отошёл, не беспокоить) в выпадающем списке.

### Настройки подключения

- Перейдите на страницу `/settings`.
- Укажите URL сервера (если приложение развёрнуто на другом домене) и учётные данные для API.

### Админ-панель

Доступна по адресу `/admin` для пользователей с ролью `ADMIN` или `SUPER_ADMIN`.

- **Пользователи**: Просмотр, добавление, редактирование, деактивация пользователей.
- **Иерархия компании**: Создание и редактирование отделов, назначение руководителей.
- **Экспорт истории**: Выгрузка всех сообщений за выбранный период в формате CSV.
- **Очистка истории**: Удаление старых сообщений (требуются права супер-администратора).

## Мобильное приложение

1. Установите Capacitor:
   ```bash

   ```

2. Добавьте платформу Android:
   ```bash
   npx cap add android
   ```

3. Соберите проект Next.js:
   ```bash
   npm run build
   ```

4. Скопируйте сборку в Capacitor:
   ```bash
   npx cap copy
   npx cap sync android
   ```

5. Откройте проект в Android Studio:
   ```bash
   npx cap open android
   ```


2. Скопируйте сборку:
   ```bash

   npx cap sync ios
   ```

3. Откройте проект в Xcode:

4. Соберите приложение через Xcode.

## API Endpoints

Основные API маршруты:

- `POST /api/auth/register` – регистрация (только для админов)
- `POST /api/auth/login` – вход
- `GET /api/chats` – список чатов
- `POST /api/chats` – создание чата
- `GET /api/messages?chatId=...` – сообщения чата
- `POST /api/messages` – отправка сообщения
- `PATCH /api/messages/[id]` – редактирование сообщения (в течение 5 секунд)
- `POST /api/messages/forward` – пересылка сообщения
- `POST /api/messages/broadcast` – рассылка сообщений
- `POST /api/upload` – загрузка файла
- `GET /api/admin/users` – управление пользователями (админ)
- `POST /api/admin/export` – экспорт истории (админ)
- `GET /api/cron/birthday-reminders` – отправка уведомлений о днях рождения (защищён секретным ключом)

Подробнее смотрите в исходном коде.

## Локальное хранение файлов

Все загруженные файлы сохраняются в `public/uploads`. Каждому файлу присваивается уникальное имя (UUID) для предотвращения конфликтов. Файлы доступны по URL вида `http://localhost:3000/uploads/{filename}`.

Для продакшена рекомендуется настроить CDN или облачное хранилище (например, AWS S3), изменив логику в `app/api/upload/route.ts`.

## Настройка уведомлений о днях рождения

Для автоматической отправки предупреждений за два дня до дня рождения пользователя используется endpoint `/api/cron/birthday-reminders`. Он защищён секретным ключом, который должен быть передан в заголовке `Authorization: Bearer <CRON_SECRET>`.

### Настройка переменных окружения

Добавьте в `.env.local`:

```
CRON_SECRET="your-cron-secret-key"
```

### Планировщик (Cron)

Настройте ежедневный вызов этого endpoint в 9:00 утра (или в любое другое время) с помощью:

- **Vercel Cron Jobs** (если развёртываете на Vercel) – добавьте в `vercel.json`:
```json
{
  "crons": [
    {
      "path": "/api/cron/birthday-reminders",
      "schedule": "0 9 * * *"
    }
  ]
}
```
- **Другой сервис** (например, GitHub Actions, cron на сервере) – используйте curl:
```bash
curl -H "Authorization: Bearer your-cron-secret-key" https://your-domain.com/api/cron/birthday-reminders
```

### Логика работы

1. Endpoint получает всех пользователей с заполненным полем `birthDate`.
2. Фильтрует тех, у кого день рождения через 2 дня (игнорируя год).
3. Находит или создаёт системный чат "Birthday Notifications" (тип CHANNEL).
4. Отправляет в этот чат сообщение-уведомление для каждого именинника.

## Развёртывание

### Vercel

1. Подключите репозиторий к Vercel.
2. Укажите переменные окружения в настройках проекта.
3. Добавьте команду сборки:
   ```bash
   prisma generate && prisma db push && next build
   ```
4. Деплой.

### Docker

Создайте `Dockerfile`:

```dockerfile
FROM node:18-alpine
WORKDIR /app
COPY package*.json ./
RUN npm install
COPY . .
RUN npx prisma generate
RUN npm run build
EXPOSE 3000
CMD ["npm", "start"]
```

Соберите и запустите:
```bash
docker build -t business-chat .
docker run -p 3000:3000 --env-file .env.local business-chat
```
